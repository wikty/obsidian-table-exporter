# Security Policy

## Reporting a Vulnerability

If you believe you have found a security issue in `Obsidian Table Exporter`, please do not post full exploit details in a public issue.

Instead:

1. Open a private report if GitHub private vulnerability reporting is enabled for the repository.
2. If that is not available, contact the maintainer through the repository profile and include:
   - a clear description of the issue
   - reproduction steps
   - affected plugin version
   - impact assessment, if known

## Scope

Relevant reports include issues such as:

- arbitrary file access beyond expected plugin behavior
- unintended command execution
- unsafe handling of exported content
- dependency vulnerabilities that materially affect users

## Supported Versions

The latest released version will receive priority for security fixes. Older unreleased local builds may not be supported.
